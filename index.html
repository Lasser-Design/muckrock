Labels

Paragraphs (p) can be assigned color classifications, turning them into labels. These labels, through the color hierarchy, carry tones and implicit messages to facilitate communication. The color categories are added through corresponding classes: default, primary, success, failure, and disabled.


<p class="default">This is a default label.</p>
<p class="primary">Hello. This is a primary label. I am important!</p>
<p class="success">Great job! This is a success label. I am good!</p>
<p class="failure">Oh no! This is a failure label. I am bad!</p>
<p class="disabled">Nothing to see here. This is a disabled label.</p>

This is a default label. labels deliver information keeping with a specific tone. I am neutral.

Hello. This is a primary label. I am important!

Great job! This is a success label. I am good!

Oh no! This is a failure label. I am bad!

Nothing to see here. This is a disabled label.

Panels

A panel is an object with the class panel and an optional color classifier. Without a color classifier, the panel defaults to the default classification. The available classifications are the previously established default, primary, success, failure, and disabled.


<object class="panel">
  <p>Just an ordinary panel.</p>
</object>

<object class="primary panel">
  <p>I am a primary panel. Pay attention to me!</p>
</object>
Buttons

Buttons are a tags with the class button. A button without one of the display classes [default, primary, success, failure, disabled] is styled the same as a default button. Disabled buttons are span tags.


<a class="primary button" href="#">Primary</a>
<a class="success button" href="#">Success</a>
<a class="failure button" href="#">Failure</a>
<a class="default button" href="#">Default</a>
<span class="disabled button" href="#">Disabled</span> Primary Success Failure Default Disabled

Inputs & Feedback

Progress bars are objects with the class progress-bar and they contain a span with the class meter. The width of the meter is set within the HTML markup and available to manipulate with Javascript. When a meter has a width of 100%, it must have the completed class. The meter can optionally contain a color classifier class, like success or failure.


<object class="progress-bar">
  <span class="meter" style="width: 30%;"></span>
</object>
<object class="progress-bar">
  <span class="failure meter" style="width: 60%;"></span>
</object>
<object class="progress-bar">
  <span class="completed success meter" style="width: 100%;"></span>
</object>


Text inputs are customized, in order to match the typographic and formal heirarchy of the rest of the design. Button inputs are left as they are. Forms should be grouped within fieldset tags, with a label preceeding the input. There is a special fieldset class, login, since these are the most important fieldset. Inputs can be set as required or disabled, and they can be given any of the established color classes: primary, success, failure, default.


<fieldset class="login">
  <fieldset>
    <label>Username</label>
    <input type="text" size="15" required>
  </fieldset>
  <fieldset>
    <label>Password</label>
    <input type="password" size="15" required>
  </fieldset>
  <fieldset>
    <input type="submit" class="primary button" value="Log In" />
    <input type="submit" class="default button" value="Sign Up" />
  </fieldset>
</fieldset>

<fieldset>
  <label>Text</label>
  <input type="text" size="20" placeholder="Blah blah blah" />
</fieldset>

<fieldset>
  <label>Email (Required)</label>
  <input type="email" size="20" placeholder="example@muckrock.com" required/>
</fieldset>
